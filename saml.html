SAML-like デモ

SAML-likeフローの基本要素と検証ポイントを、ブラウザだけで体験します。

このデモで学ぶこと

  • SP Entity ID / ACS URL / 共有鍵などの役割
  • AuthnRequest 〜 Assertion 生成までの流れ
  • 署名検証やAudience確認などの検証ポイント

このデモを初期化

SAML-likeデモ用のアプリ設定とユーザーを投入します。

次に進む

Adminへ SPでログイン開始 IdPへ

おすすめ手順

  1. AdminでSAML-likeアプリ設定を確認
  2. SPでログイン開始（IdPへリダイレクト）
  3. IdPでユーザー認証
  4. IdPがアサーション生成してSPへ送信
  5. SPのACSが検証してセッション作成
